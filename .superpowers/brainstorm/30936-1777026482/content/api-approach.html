How should Next.js talk to the FastAPI backend?

This affects deployment, auth, and how API calls are made throughout the app

A

Next.js API Routes as proxy (Recommended)

Next.js has its own /api/* routes that forward calls to FastAPI. The browser never talks to FastAPI directly. Cleaner CORS, one deployment URL, easy to add auth middleware later.

B

Direct API calls from browser

React components call http://localhost:8000 directly. Simpler for local dev, but requires CORS config on FastAPI and exposes the backend URL to the browser. Fine for MVP.

C

Server-side only (Next.js Server Components)

All FastAPI calls happen server-side in Next.js Server Components or getServerSideProps. Browser never touches FastAPI. Most secure, but more complex state management.
