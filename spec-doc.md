## Part 1 — Product Requirements

### What is this?

A business health monitoring system for Indian MSMEs. It collects data from three sources — Google Places (reviews + competitors), and synthetic POS sales data — runs a scoring algorithm, generates three specific AI-powered insights using Claude API, and returns a structured JSON report via a FastAPI backend.

### Who is it for?

Primary user (for now): the developer (Pratham) running reports manually for 5–10 beta users. The MSME owner receives the report via WhatsApp from Pratham manually — there is no owner-facing UI in MVP or V1.

### What problem does it solve?

Indian MSME owners operate blind — no unified view of how their business is performing relative to their reviews, competitors, and their own sales trends. VyaparAI gives them a weekly 0–100 health score with three specific actions. It replaces a ₹30,000/month business consultant at ₹999/month.

### What does it do — feature by feature?

**Business onboarding**
- Developer registers a business via `POST /onboard`
- Inputs: business name, Google Maps Place ID, category (restaurant / retail / manufacturing), owner name
- System validates the Place ID against Google Places API
- Returns a UUID for that business — used in all subsequent calls

**Google data collection**
- Pulls: star rating, total review count, last 5 reviews (text + rating + relative time)
- Pulls: up to 10 nearby competitors within 800m, sorted by rating
- Stores raw data in `reviews_cache` table with timestamp
- Runs once per report generation — not continuously

**POS data ingestion**
- Developer uploads a CSV file via `POST /upload-pos/{business_id}`
- CSV contains: date, product_category, units_sold, revenue, transaction_count, avg_order_value
- System validates columns, converts date format, bulk inserts into `pos_records`
- For MVP: CSV is synthetic data generated by `generate_synthetic_pos.py`
- For V1: CSV is a real export from Tally / Excel / Petpooja

**POS signal computation**
- Computed from last 30 days of `pos_records`
- Signals: 30-day revenue trend (% vs prior 30 days), slow-moving categories (units < 30% avg for 14 days), top product by revenue, AOV trend direction
- If no POS data exists for a business: all signals return None, pos_score defaults to 50 (neutral)

**Health score calculation**
- Three sub-scores combined into one final score (0–100):
  - Review score (40% weight): rating quality + volume + recent trend
  - Competitor score (25% weight): business rating vs avg competitor rating within 800m
  - POS score (35% weight): revenue trend + slow inventory + AOV health
- Final score: integer 0–100
- Score bands: 80+ = healthy (green), 60–79 = watch (yellow), below 60 = at risk (red)

**AI insight generation**
- Claude API (`claude-sonnet-4-20250514`) generates exactly 3 insights + 1 weekly action
- Input to Claude: business name, final score, Google rating, review count, last 3 review snippets, top 3 competitor names + ratings, revenue trend %, slow category names, AOV direction
- Output format: strict JSON `{"insights": ["...", "...", "..."], "action": "..."}`
- Quality bar: insights must name specific products and specific competitor names — not generic advice
- Language: English only for MVP and V1

**Report retrieval**
- `POST /generate-report/{business_id}` runs the full pipeline and saves to `health_scores` table
- `GET /history/{business_id}` returns last 12 weekly scores ordered by date
- Both endpoints return JSON — no UI in MVP

### What does a good report look like?

```json
{
  "business_id": "uuid-here",
  "business_name": "Sharma Tiffin Centre",
  "final_score": 71,
  "sub_scores": {
    "review_score": 78,
```

This is the north star. Every prompt iteration judges itself against producing output this specific.

### What does NOT exist in MVP?

- No WhatsApp delivery (manual send by developer)
- No Monday scheduler (manual trigger via API)
- No Hindi language
- No frontend / dashboard / UI of any kind
- No authentication on the API
- No payment processing
- No ML forecasting
- No multi-agent system
- No Zomato / Swiggy scraping
- No real POS API integration (synthetic CSV only)

### Health score formula

```python
# Sub-scores (each 0-100)
review_score = int(
    (rating / 5.0) * 55 +                          # rating quality: 0-55
    min(25, math.log10(max(total_reviews, 1)) * 10) +  # volume: 0-25
    (avg_recent_rating / 5.0) * 20                  # recent trend: 0-20
)

competitor_score = int(clamp(
    60 + (my_rating - mean(competitor_ratings)) * 30, 0, 100
))
# No competitors nearby → 65 neutral

pos_score = int(
    revenue_trend_pts +     # 0-50: positive trend = 50, -30%+ = 0
    inventory_pts +         # 0-30: 0 slow categories = 30, 3+ = 0
    aov_pts                 # 0-20: rising AOV = 20, falling = 5
)
# No POS data → 50 neutral

final_score = int(
    review_score * 0.40 +
    competitor_score * 0.25 +
    pos_score * 0.35
)
```

### Claude prompt structure

```
System: You are a concise business advisor for Indian MSME owners. 
Give specific, actionable advice. Always name specific products and 
competitors. Never give generic advice. Actions must cost under 
₹2,000 and take under 3 hours.

User: Business: {name}
Health score: {final_score}/100
Google rating: {rating} ({total_reviews} reviews)
Last 3 reviews: {review_snippets}
Top competitors: {competitor_names_and_ratings}
Revenue trend: {trend_pct}% vs last month
Slow-moving: {slow_categories}
AOV trend: {aov_direction}

Return ONLY valid JSON with no markdown:
{"insights": ["...", "...", "..."], "action": "..."}
```

### Error handling rules

- Google API timeout → retry once after 2s → if fails, raise `GoogleAPIError`
- Invalid place_id → raise `ValueError` with message, return 400
- Claude JSON parse failure → retry once with stricter prompt → if fails, log and return 500
- POS upload with wrong columns → return 422 with list of missing columns
- One business failing in a batch → log error, continue loop, never crash pipeline

### Quality gates before V1

- 10 Claude outputs rated by Pratham — must average ≥ 3.5/5 on specificity
- Pipeline runs for 10 consecutive businesses without unhandled exceptions
- Healthy synthetic profile scores 75+, struggling profile scores below 40
- `pos_signals()` correctly identifies the deliberately slow category in test data

### Infrastructure

- Local development only for MVP (no deployment)
- Supabase free tier: 500MB, 50K rows — sufficient for 500 businesses × 52 weeks
- Google Places API: $200 free credit/month ≈ 40,000 place lookups
- Anthropic API: ~₹100–200 per user per month at 1 report/week
